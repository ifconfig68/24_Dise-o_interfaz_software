Ejemplos rendimiento WPO
Reducir peticiones HTTP Objetivó al realizar menos peticiones http es reducir el tiempo de descarga

COMBINAR VARIOS CSS O JS EN UNO Categoría: Conectividad, CSS, JavaScript
Problemas: la descarga en paralelo de estos elementos hace que se puedan bloquear otros. Uno de los mayores costes de conectividad es la propia conexión y desconexión de las descargas. Solución tener un único fichero .JS o .CSS de un tamaño mayor que no varios pequeños usando HttpCombiner

COMBINAR IMÁGENES O ICONOS EN CSS SPRITES Categoría: Conectividad, CSS, Imágenes
Problema iconos que se van repitiendo a lo largo de las distintas páginas. Produce efecto de un exceso de peticiones y, por ello, un número elevado de conexiones inútiles. Solución integrar varias imágenes en una imagen única. Esta imagen, que suele ser un PNG8, contiene una cantidad de imágenes pequeñas suficiente para no tener un tamaño excesivo y con una única imagen poder gestionar todos los elementos desde el CSS Ejemplo #nav li a {background-image:url('imagen.png')}

PARALELIZAR PETICIONES HTTP
Protocolo que permite el intercambio de información, que principalmente es el HTTP. es muy recomendable la combinación de CSS y JS o de las imágenes en CSS Sprites. es mejor hacer una petición grande que muchas pequeñas para reducir el coste de estas comunicaciones.

INCLUIR LOS JS EN LA PARTE INFERIOR Categoría: Conectividad, JavaScript
se puede llegar a hacer bloquear todo el sitio si por alguna razón el JavaScript no puede procesarse en un tiempo razonable, y en tal caso puede bloquear imágenes, CSS u otros contenidos. se recomienda que, en la medida de lo necesario, los ficheros de JavaScript se sitúen al final de la página, para que se puedan cargar todos los otros elementos antes, y al final estos scripts
